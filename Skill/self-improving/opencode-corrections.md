# 自我改进修正日志

> 追踪用户明确的修正和可复用的经验教训。
> 格式: 带时间戳的 Markdown。
> 整理: 50条后，归档最旧的25条。

---

## 条目格式

```markdown
### YYYY-MM-DD HH:MM — [分类]

**修正:** "..."
**上下文:** 触发原因
**计数:** N (3次后升级到 memory.md)
**状态:** 暂定 | 待确认 | 已确认 | 已升级
```

## 分类

- `code`: 代码风格、模式、架构
- `communication`: 语气、格式、冗长
- `workflow`: 流程、方法、工具
- `project`: 项目特定覆盖

## 示例

```markdown
### 2026-03-24 10:30 — communication

**修正:** "别以 '当然' 或 '好的' 开头"
**上下文:** 通用聊天回复
**计数:** 1
**状态:** 暂定
```
